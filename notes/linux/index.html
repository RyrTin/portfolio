Linux 笔记

记录 Linux 开发环境配置、日常使用技巧与服务器运维经验。以实用为导向，覆盖 WSL2、Shell 脚本、性能监控等方向。

第一章 WSL2 & Ubuntu 开发环境搭建

1. WSL2 是什么、为什么用

WSL2（Windows Subsystem for Linux 2）是 Windows 内置的 Linux 虚拟化方案，相比虚拟机更轻量，文件系统互通，能直接跑 Linux 二进制程序。

对我而言最实用的几点：

  • /mnt/c/ 挂载 C 盘，Linux 和 Windows 文件系统直接互通
  • 能跑 Docker，不需要 Docker Desktop 的额外开销
  • VSCode 的 WSL 远程插件体验接近原生
  • 比完整虚拟机省内存，启动秒级
⚠️ 如果机器是 Windows 10 且版本较老（19041 之前），建议先升级。WSL2 对内核版本有要求，老版本装起来很折腾。

2. WSL2 安装（PowerShell 一行搞定）

以管理员身份打开 PowerShell，执行：

wsl --install

这条命令会自动完成：启用 WSL 可选功能、安装 WSL2 内核、设为默认版本、安装 Ubuntu 作为默认发行版。

安装完成后重启电脑，首次进入 Ubuntu 会要求设置用户名和密码（输入密码时终端不显示字符，正常的）。重启后在开始菜单找 "Ubuntu" 或在 PowerShell 里输入 wsl 进入。

💡 如果提示"无法解析服务器地址"或卡在下载，大概率是网络问题。可从 Microsoft 官方文档 手动下载离线包安装。

3. Ubuntu 基础配置

3.1 换国内源（apt 加速）

Ubuntu 默认源在国外，apt update 经常超时。换成清华源：

# 备份原文件 sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak # 替换为清华源（Ubuntu 22.04 / jammy） sudo sed -i "s@http://.*archive.ubuntu.com@https://mirrors.tuna.tsinghua.edu.cn@g" /etc/apt/sources.list sudo sed -i "s@http://.*security.ubuntu.com@https://mirrors.tuna.tsinghua.edu.cn@g" /etc/apt/sources.list # 更新索引 sudo apt update && sudo apt upgrade -y

Ubuntu 24.04（Noble）的 sources.list 格式变了（DEB822），直接编辑 /etc/apt/sources.list.d/ubuntu.sources 改 URL。

3.2 安装基础工具包

sudo apt install -y \ build-essential \ curl wget git \ vim tmux \ htop tree \ net-tools iproute2 \ software-properties-common \ ca-certificates gnupg lsb-release

这几个包覆盖了 90% 的日常需求：编译工具链、下载、Git、编辑器、进程监控、网络诊断。

3.3 配置 Git

git config --global user.name "你的名字" git config --global user.email "你的邮箱" git config --global core.editor "vim" git config --global pull.rebase false

4. Windows 与 WSL 文件系统互通

4.1 从 WSL 访问 Windows 文件

Windows 各盘符自动挂载在 /mnt/ 下：

cd /mnt/c/Users/你的用户名/ code /mnt/c/Users/你/Desktop/test.py

4.2 从 Windows 访问 WSL 文件

在文件资源管理器地址栏输入 \\wsl$\Ubuntu\home\用户名\，或在 WSL 里输入 explorer.exe . 直接打开当前目录。

4.3 坑：跨文件系统性能差异

在 /mnt/c/ 下跑 Linux 工具（pip install、npm install）会比 WSL 原生文件系统慢 5-10 倍。

💡 最佳实践：开发项目放在 ~/projects/（WSL 原生），不要用 Windows 文件系统跑 Linux 工具链。VSCode 用 Remote WSL 插件打开项目，不要直接打开 \\wsl$\... 路径。

5.1 Python 环境（pyenv + venv）

5.2 VSCode + WSL 远程开发

在 Windows 侧 VSCode 安装 WSL 扩展，然后在 WSL 终端里输入 code .，VSCode 会自动连接 WSL 环境。左下角显示 "WSL: Ubuntu" 即连接成功，终端/解释器/linter 全部在 WSL 里跑。

6. 常见问题 & 踩坑记录

WSL2 占用内存过高

WSL2 默认会吃满宿主机内存。在 %USERPROFILE%\.wslconfig（Windows 侧）限制：

[wsl2] memory=8GB swap=4GB localhostForwarding=true

修改后执行 wsl --shutdown 重启生效。

apt update 时 GPG 密钥错误

现代 Ubuntu（22.04+）正确处理方式——不要用已废弃的 apt-key：

curl -fsSL https://example.com/KEY.gpg | sudo gpg --dearmor -o /etc/apt/keyrings/example.gpg echo "deb [signed-by=/etc/apt/keyrings/example.gpg] https://example.com/ stable main" \ | sudo tee /etc/apt/sources.list.d/example.list

WSL IP 每次重启后变化

WSL2 是 NAT 网络，IP 会变。访问 WSL 里的服务直接用 localhost，WSL2 会自动端口转发。如果转发失效，确认 .wslconfig 里有 localhostForwarding=true。

基础导航

2. 文件查看与编辑

cat file.txt # 全量输出 less file.log # 分页浏览（q 退出、/搜索、n 下一处） head -n 20 file # 前 20 行 tail -f app.log # 实时追踪末尾 wc -l file # 统计行数 diff a.txt b.txt # 对比差异 vim file # 编辑器
💡 日常配 alias less="less -R" 可以让 less 正确显示带颜色的日志（如 pytest 输出）。

3. 权限管理

chmod 755 script.sh # rwxr-xr-x chmod +x script.sh # 添加执行权限 chown user:group file # 修改所有者 ls -l # 查看权限 umask 022 # 设置默认权限掩码

权限三位一组：rwx = 读(4) + 写(2) + 执行(1)。常见组合：644（文件）、755（目录/脚本）、600（私钥）。

⚠️ chmod 777 是"所有人都能读+写+执行"，除 /tmp 外几乎永远不该用。

4. 进程管理

ps aux | grep python # 查找 Python 进程 kill -9 PID # 强制杀进程（先试 kill PID） pkill -f "python train.py" # 按命令行匹配杀 htop # 交互式进程浏览器 bg / fg # 后台/前台任务切换 jobs # 列出当前 shell 的后台任务 nohup python train.py > log.txt 2>&1 & # 后台跑，关终端不中断

jobs / bg / fg 用法

python train.py & # 后台启动 Ctrl+Z # 暂停当前前台任务 bg %1 # 让 job 1 在后台继续 fg %1 # 让 job 1 回到前台 disown -h %1 # 从 shell 的 job table 移除，关终端不被杀
💡 在服务器跑长时间训练任务：优先用 nohup 或 tmux，比 disown 更可靠。

5. 网络诊断

ping -c 4 google.com # 连通性测试（发 4 个包） curl -I https://api.example.com # 只返回 HTTP 头 curl -X POST -H "Content-Type: application/json" -d '{"k":"v"}' url wget -c url # 断点续传下载 ss -tlnp # 查看监听端口 ip addr / ifconfig # 查看 IP 地址 nc -zv host port # 测试端口是否通 dig example.com # DNS 查询 lsof -i :8080 # 查看谁占用了 8080 端口

6. 文本处理三剑客

grep — 搜索过滤

grep "error" app.log # 基础搜索 grep -r "TODO" src/ # 递归搜索目录 grep -i "warning" log # 忽略大小写 grep -v "DEBUG" log # 排除匹配行 grep -A 3 "error" log # 显示匹配行及后 3 行 grep -c "200" access.log # 只输出匹配行数 grep -E "err(or|ing)" log # 扩展正则

sed — 流编辑器（替换神器）

sed 's/foo/bar/' file # 替换每行第一个 foo sed 's/foo/bar/g' file # 全局替换 sed -i 's/old/new/g' file # 原地修改（⚠️ 先备份） sed '/pattern/d' file # 删除匹配行 sed -n '5,10p' file # 只打印第 5-10 行

第三章 Shell 脚本与自动化

1. 基础语法速览

Shebang 与变量

#!/bin/bash # 上面这行告诉系统用哪个解释器 # 变量（等号两边不能有空格！） NAME="world" echo "Hello, $NAME" # Hello, world echo "Hello, ${NAME}_suffix" # 用花括号避免歧义 # 命令替换 NOW=$(date +%Y-%m-%d) FILES=$(ls *.txt | wc -l) # 特殊变量 echo "Script: $0" echo "Arg1: $1, Arg2: $2" echo "All args: $@" echo "Arg count: $#" echo "Exit code: $?" # 上一条命令的返回值

字符串操作

STR="hello-world.py" echo ${STR%.py} # hello-world (去最短后缀) echo ${STR##*.} # py (去最长前缀) echo ${STR/-/_} # hello_world.py (替换第一个) echo ${STR//-/ } # hello world.py (替换全部) echo ${#STR} # 15 (字符串长度)
💡 变量一定要加引号！rm -rf $DIR/（若 $DIR 为空）= rm -rf /。改写成 rm -rf "$DIR/" 或 rm -rf "${DIR:?}" 安全得多。

2. 条件判断与循环

if 语句

# 文件测试 if [ -f "config.yaml" ]; then echo "config found" elif [ -d "data/" ]; then echo "data dir exists" else echo "nothing found" fi # 数值比较（中括号内两侧必须有空格！） if [ "$COUNT" -gt 10 ]; then echo "count > 10" fi # 字符串比较 if [ "$MODE" = "train" ]; then echo "training mode" fi # 命令成败判断 if command -v python3 &> /dev/null; then echo "python3 available" fi

for / while 循环

# 遍历列表 for seed in 42 123 456; do python train.py --seed "$seed" done # 遍历文件 for config in configs/*.yaml; do python run.py --config "$config" done # while 循环（逐行读文件） while IFS= read -r line; do echo "-> $line" done < input.txt # 无限循环 + 检测条件 while true; do sleep 3600 python check_gpu.py || break done

3. 函数与参数

# 定义 log() { local level=$1 # local 限制作用域 local msg=$2 echo "[$(date '+%H:%M:%S')] [$level] $msg" } log "INFO" "training started" # 返回值（只能 0-255，0=成功） check_gpu() { nvidia-smi &> /dev/null return $? # 返回 nvidia-smi 的退出码 } if check_gpu; then echo "GPU available" fi # 函数输出捕获 get_gpu_count() { nvidia-smi -L | wc -l } GPU_COUNT=$(get_gpu_count) echo "Found $GPU_COUNT GPU(s)"

4. 实用脚本案例

案例 1：批量超参数实验

#!/bin/bash # run_experiments.sh — 遍历学习率跑实验 LR_LIST=(1e-3 5e-4 1e-4 5e-5) SEEDS=(42 123 456) for lr in "${LR_LIST[@]}"; do for seed in "${SEEDS[@]}"; do echo "=== lr=$lr seed=$seed ===" python train.py \ --lr "$lr" \ --seed "$seed" \ --output_dir "outputs/lr${lr}_s${seed}" done done

案例 2：日志清理脚本

#!/bin/bash # clean_logs.sh — 删除 N 天前的日志并报告大小 DAYS=${1:-30} # 默认 30 天 DIR=${2:-logs} echo "Cleaning files older than $DAYS days in $DIR..." SIZE_BEFORE=$(du -sh "$DIR" 2>/dev/null | cut -f1) find "$DIR" -name "*.log" -mtime +"$DAYS" -delete SIZE_AFTER=$(du -sh "$DIR" 2>/dev/null | cut -f1) echo "Done. $DIR: $SIZE_BEFORE → $SIZE_AFTER"

案例 3：检查 GPU 空闲后自动跑训练

#!/bin/bash # wait_and_train.sh — 等待 GPU 显存低于阈值后启动训练 THRESHOLD_MB=${1:-2000} # 默认 2GB while true; do FREE=$(nvidia-smi --query-gpu=memory.free --format=csv,noheader,nounits | head -1) if [ "$FREE" -gt "$THRESHOLD_MB" ]; then echo "[$(date)] GPU free: ${FREE}MB, starting training" python train.py --epochs 100 break fi echo "[$(date)] GPU free: ${FREE}MB < ${THRESHOLD_MB}MB, waiting..." sleep 60 done

第五章 SSH 远程开发与服务器运维

1. SSH 基础与密钥登录

生成密钥对并部署

# 本机生成 ED25519 密钥（比 RSA 更快更安全） ssh-keygen -t ed25519 -C "your_email@example.com" # 一路回车即可，私钥: ~/.ssh/id_ed25519 # 把公钥复制到服务器 ssh-copy-id user@server_ip # 或手动追加 cat ~/.ssh/id_ed25519.pub | ssh user@server_ip \ "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys" # 在服务器上修改权限（必须！） chmod 700 ~/.ssh chmod 600 ~/.ssh/authorized_keys

安全加固（服务端）

# 编辑 /etc/ssh/sshd_config PermitRootLogin no # 禁止 root 直接登录 PasswordAuthentication no # 禁用密码登录（仅用密钥） PubkeyAuthentication yes # 启用密钥认证 Port 2222 # 换非标准端口（减少扫描） # 重启 SSH 服务 sudo systemctl restart sshd
⚠️ 改端口前务必确认密钥登录正常，否则改完把自己锁在外面。

2. SSH Config — 简化连接

第六章 性能监控与调优

1. CPU 与内存监控

htop — 交互式进程浏览器

sudo apt install htop -y htop # F6 选择排序依据（CPU%、MEM%、TIME） # F9 杀进程 # F5 切换成树状视图（看父子进程关系）

对比 top：htop 彩色、可鼠标点击、可直接杀进程、横竖滚动都支持，实用性强得多。

内存概况

free -h # 内存总量、使用量、可用量 vmstat 1 5 # 每秒刷新一次，共 5 次 # si/so 列：如果不是 0，说明在用 swap（内存不够了）

2. GPU 监控 (nvidia-smi)

基础查询

nvidia-smi # 一次性查看全部 GPU 状态 nvidia-smi -l 1 # 每秒刷新（Ctrl+C 停止） watch -n 1 nvidia-smi # 同上，watch 也可以 nvidia-smi -q -d MEMORY # 只查显存详情

检查 GPU 被谁占用

# 列出所有 GPU 上的进程 nvidia-smi pmon -c 1 # 每 1 秒快照 fuser -v /dev/nvidia* # 列出所有访问 GPU 的进程

gpustat — 更清爽的 nvidia-smi

pip install gpustat gpustat -i 1 # 每秒刷新，彩色显示显存使用条 gpustat -cp # 显示每个 GPU 上的进程 watch --color -n 1 gpustat --color

gpustat 输出比 nvidia-smi 紧凑得多，一眼看清哪些 GPU 空闲、谁在用什么。

第七章 Docker 基础与实践

1. 核心概念

Docker 用"容器"打包应用及其所有依赖，确保在任何机器上运行一致。三个核心概念：

  • 镜像（Image）：只读模板，含 OS 文件、依赖库、代码。从 Dockerfile 构建。
  • 容器（Container）：镜像的运行实例，轻量级沙箱，共享宿主机内核。
  • 仓库（Registry）：镜像的存储和分发中心（Docker Hub / 阿里云镜像仓库）。

2. 常用命令

# 镜像 docker pull python:3.11-slim # 拉取镜像 docker images # 列出本地镜像 docker rmi image_id # 删除镜像 # 容器 docker run -it --rm python:3.11 bash # 启动交互式容器 docker run -d --name myapp -p 8080:80 nginx # 后台运行，端口映射 docker ps # 运行中的容器 docker ps -a # 所有容器 docker logs -f myapp # 实时查看日志 docker exec -it myapp bash # 进入运行中的容器 # 清理 docker system prune -a # 清理所有未用的镜像/容器/网络

docker run 常用参数速查

docker run \ -d --name myapp \ # 后台运行 + 命名 --gpus all \ # 挂载所有 GPU -p 8080:80 \ # 端口映射 -v /host/data:/app/data \ # 挂载数据卷 -e ENV_VAR=value \ # 环境变量 --shm-size=8g \ # 共享内存大小（PyTorch 需要） --restart=unless-stopped \ # 崩溃自动重启 image:tag

3. Dockerfile 编写

Python 项目示例

FROM python:3.11-slim WORKDIR /app # 先复制依赖文件（利用 Docker 层缓存） COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt # 再复制代码 COPY . . EXPOSE 8000 CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

PyTorch 项目示例

FROM pytorch/pytorch:2.1.0-cuda12.1-cudnn8-runtime WORKDIR /workspace COPY requirements.txt . RUN pip install --no-cache-dir -r requirements.txt COPY . . ENTRYPOINT ["python", "train.py"]
💡 先 COPY requirements.txt 再 COPY . 是关键优化——依赖文件变动少，Docker 会复用缓存层。

4. Docker Compose

当项目涉及多个容器（如 API + Redis + PostgreSQL），用 Compose 一把管理：

# docker-compose.yml version: "3.8" services: api: build: . ports: - "8000:8000" volumes: - ./data:/app/data environment: - REDIS_URL=redis://redis:6379 - DATABASE_URL=postgresql://postgres:postgres@db:5432/mydb depends_on: - redis - db redis: image: redis:7-alpine db: image: postgres:16-alpine environment: POSTGRES_PASSWORD: postgres POSTGRES_DB: mydb
docker compose up -d # 后台启动所有服务 docker compose down # 停止并删除容器 docker compose logs -f api # 查看 api 服务的日志 docker compose build # 重新构建镜像

5. 深度学习容器化实践

把训练环境 Docker 化的几个实际好处：

  • 论文复现仓库自带 Dockerfile → 一行命令获得完全相同的环境
  • 多台 GPU 服务器部署同一训练任务，不用担心 CUDA/cuDNN 版本差异
  • 实验环境隔离，Python 包冲突再也跟你没关系

常用基础镜像选择：

# 官方 PyTorch 镜像（推荐） pytorch/pytorch:2.1.0-cuda12.1-cudnn8-runtime pytorch/pytorch:2.1.0-cuda12.1-cudnn8-devel # 需编译 CUDA 扩展时 # NVIDIA 官方 CUDA 镜像 nvidia/cuda:12.1.0-cudnn8-runtime-ubuntu22.04

docker-compose 训练配置示例

# docker-compose.train.yml services: train: image: pytorch/pytorch:2.1.0-cuda12.1-cudnn8-runtime volumes: - ./:/workspace - /data/datasets:/datasets:ro # 数据集只读挂载 command: python train.py --epochs 100 --batch-size 64 deploy: resources: reservations: devices: - driver: nvidia count: 1 capabilities: [gpu] shm_size: "8gb" environment: - CUDA_VISIBLE_DEVICES=0
